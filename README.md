# awg-relay

Релей между клиентом и VPN-сервером. Нужен, когда трафик до сервера режут
из-за его хостинга. Ставишь впереди дешёвый российский сервер — он принимает
твоё подключение и пересылает на основной. Шифрование не трогается, релей
просто перекидывает пакеты.

```
клиент → релей (РФ) → основной сервер (Amnezia)
```

## Файлы

- `setup-relay.sh` — ставится на релей, настраивает пересылку портов.
- `patch-client.sh` — меняет в твоём конфиге адрес сервера на адрес релея.

## Как настроить

**1. Основной сервер.** Настрой Amnezia как обычно.

**2. Экспорт.** В Amnezia: Share → AmneziaWG → файл. Запомни порт из строки
`Endpoint`.

**3. Релей.** Зайди по SSH на российский сервер и выполни:
```bash
sudo ./setup-relay.sh add --name main --dest <IP_СЕРВЕРА> --dport <ПОРТ>
```

**4. Конфиг клиента.** На своей машине:
```bash
./patch-client.sh --in <файл.conf> --relay <IP_РЕЛЕЯ>
```
Получится новый файл `*-relay.conf`.

**5. Импорт.** Импортируй его в Amnezia как новое подключение и подключись.

**6. Проверка.** Включи VPN, запусти `mtr <IP_РЕЛЕЯ>` и параллельно что-нибудь
скачай. Если потери близко к нулю — всё работает.

## Команды релея

```bash
sudo ./setup-relay.sh add --name N --dest IP --dport PORT   # добавить пересылку
sudo ./setup-relay.sh list                                  # показать все
sudo ./setup-relay.sh remove --name N                       # убрать одну
sudo ./setup-relay.sh reapply                               # восстановить после ребута
sudo ./setup-relay.sh purge                                 # снести всё
```

Несколько серверов работают через один релей — добавляй каждый своей командой
`add` с разными `--name` и `--lport`.

Для второго протокола (например Reality по TCP):
```bash
sudo ./setup-relay.sh add --name reality --dest IP --dport 4443 --proto tcp
```

## Если рвётся соединение

Скрипт уже поднимает таймауты на релее, а патчер ставит keepalive 15 секунд —
это держит связь живой через релей. Если всё равно рвётся, проверь, что keepalive
в конфиге действительно 15, и что на релее прошла команда `reapply` после ребута.
