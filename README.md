# awg-relay

Цепочка `клиент → релей (чистый AS) → backend (Amnezia/AmneziaWG)`.

ТСПУ душит трафик к флагнутому AS, на котором стоит backend. Релей на чистом
российском AS ставится впереди: ТСПУ видит лишь «абонент → российский IP» (норма),
а хоп «релей → backend» идёт сервер-сервер и к абоненту не привязан.

Шифрование остаётся end-to-end клиент↔backend. Релей — тупой форвардер пакетов,
крипту не трогает. Несколько backend'ов работают через один релей параллельно
(каждый на своём входном порту).

```
клиент (Amnezia) --:LPORT--> релей (DNAT) --:DPORT--> backend (awg2)
```

## Скрипты

### setup-relay.sh — на релее (root)
Маршруты именованные, добавляются и удаляются независимо. Идемпотентно.
systemd-unit восстанавливает все маршруты после ребута.

```bash
# добавить маршрут
sudo ./setup-relay.sh add --name <ИМЯ> --dest <BACKEND_IP> --dport <ПОРТ>

# второй backend через тот же релей — на ДРУГОМ входном порту
sudo ./setup-relay.sh add --name <ИМЯ2> --dest <IP2> --dport <ПОРТ2> --lport <ДР_ПОРТ>

# TCP-маршрут (например, для Reality-фолбэка)
sudo ./setup-relay.sh add --name <ИМЯ> --dest <IP> --dport <ПОРТ> --proto tcp

sudo ./setup-relay.sh list            # все маршруты + правила
sudo ./setup-relay.sh remove --name <ИМЯ>
sudo ./setup-relay.sh reapply         # перечитать .route и пересобрать
sudo ./setup-relay.sh purge           # снести всё
```

Каждый маршрут помечается comment'ом `relay-managed:ИМЯ` — операции точечные,
чужой firewall не затрагивается. Вводные хранятся в `/etc/relay/<ИМЯ>.route`.

### patch-client.sh — на клиенте
Подменяет в экспортированном AmneziaWG `.conf` хост в `Endpoint` на релей.
Ключи, обфускация (Jc/S/H), AllowedIPs — без изменений.

```bash
./patch-client.sh --in <FILE> --relay <RELAY_IP> [--lport <ПОРТ>] [--out <FILE>]
```

`--lport` — когда входной порт на релее отличается от порта backend
(нужно для второго параллельного маршрута).

## Цикл

1. Backend: настроить Amnezia (поднимает `amnezia-awg2`).
2. Экспорт: Amnezia → Share → **AmneziaWG** → файл. Запомнить порт из `Endpoint`.
3. Релей: `setup-relay.sh add --name ... --dest <BACKEND_IP> --dport <ПОРТ>`.
4. Патч: `patch-client.sh --in <файл> --relay <RELAY_IP>`.
5. Импорт результата в Amnezia как НОВОЕ подключение → подключиться.
6. Проверка под нагрузкой (качать через VPN параллельно):
   `mtr <RELAY_IP>` и `mtr <BACKEND_IP>` — первый хоп чистый, потери не растут.

## Две площадки параллельно
Backend'ы независимы (свои ключи/IP). В Amnezia держишь оба профиля, переключаешь
кликом. Если оба идут через один релей — давай каждому свой `--lport` и свой
`--name`. Backend на чистом AS релея не требует — к нему подключаешься напрямую.

## Если соединение «тяжёлое»
Релей лечит деградацию по AS. Остаточные просадки под нагрузкой:
- **UDP-шейпинг** (AWG — это UDP): добавить TCP-маршрут (`--proto tcp`) и
  переключиться на Reality-профиль в Amnezia.
- **MTU**: при двойной инкапсуляции подобрать рабочий размер пакета.
- **Пинг**: базовая задержка — это расстояние до backend, релей лишь добавляет
  крюк. Снижается переносом backend ближе (под прикрытием релея можно брать
  флагнутый, но географически близкий AS).

## Безопасность
- Релей — не публичный VPN-сервис, низкий профиль.
- Сменить root-пароль backend на SSH-ключи; почистить пароль из конфигов Amnezia.
